## Система данмаку для прямых трансляций (аналог Bilibili)


## 1. Описание проекта

Проектируемая система представляет собой систему данмаку (чат сообщений) для прямых трансляций, аналогичную системе чата в Bilibili Live или Twitch.

Система обеспечивает обмен текстовыми сообщениями в реальном времени между зрителями внутри одной трансляции. Основная особенность системы — поддержка высокой нагрузки в условиях «горячих» стримов с большим количеством одновременных пользователей.

Проект не включает обработку видеопотока. В рамках работы рассматривается исключительно инфраструктура приёма, обработки, маршрутизации и доставки текстовых сообщений.

## 2. Целевая аудитория и масштаб системы

### 2.1 Целевая аудитория

Система ориентирована на пользователей стриминговой платформы:

* зрителей прямых трансляций
* стримеров
* модераторов

В качестве реального аналога используется платформа Bilibili Live.

### 2.2 Оценка проектной нагрузки

Для целей проектирования принимаются следующие параметры нагрузки:

* Одновременные пользователи (peak): 2–5 млн
* Пиковая нагрузка одной горячей трансляции: 200 000–500 000
* Оценочная частота сообщений: 300–1000 сообщений/ссообщений в секунду**

Таким образом, система должна поддерживать большое количество постоянных соединений и обеспечивать горизонтальную масштабируемость при высокой интенсивности обмена сообщениями.

---

## 3. Основной функционал системы (MVP)

### 3.1 Базовый функционал

1. Подключение пользователя к комнате трансляции
2. Установление постоянного соединения (WebSocket)
3. Отправка текстового сообщения
4. Мгновенная доставка сообщения всем участникам комнаты (fan-out)
5. Загрузка последних N сообщений при входе в комнату
6. Ограничение частоты отправки сообщений (rate limiting)
7. Базовая модерация (блокировка пользователя, фильтрация запрещённых слов)

---

## 4. Особенности highload-характера системы

Проектируемая система обладает следующими характеристиками высокой нагрузки:

* большое количество одновременных долгоживущих соединений
* высокая интенсивность входящих сообщений в популярных трансляциях
* необходимость массовой рассылки одного сообщения множеству клиентов (fan-out)
* наличие «горячих» комнат с резким ростом нагрузки

Архитектура системы будет проектироваться с учётом:

* горизонтального масштабирования
* балансировки нагрузки
* изоляции «горячих» трансляций
* отказоустойчивости
